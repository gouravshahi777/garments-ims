📊 Excel upload format

Both Purchase (Stock IN) and Sales (Stock OUT) sheets use the same format. The first row should be column headers; the rest are your transactions.

Required columns: SKU Code and Quantity.
Optional columns: Date, Note (bill no, customer name, etc).
Column order doesn't matter — match by name. SKU codes that don't exist in the system will be skipped (you'll see them in the preview).

⚡ Reorder Trigger — Sales Velocity

A SKU is flagged for reorder when X% or more of a received batch sells within Y days. This catches fast-moving items before they run out — even when current stock looks fine.

Look-back period after each purchase
Min % sold to trigger reorder
Fast movers get reordered at this × the last batch size (e.g. 2 = double)

Current rule (20 days / 75% / 2× boost): Received 4 pcs, sold 3 within 20 days → flagged as fast mover. Next order will be 4 × 2 = 8 pcs. Slow-moving items still order at normal refill quantity.

How reorder works

A SKU appears on the Reorder List if any of these are true:
1. Out of stock (current stock ≤ 0)
2. Sales velocity trigger — fast sell-through on the latest batch (settings above)
3. Below reorder point — stock ≤ Daily Sales × Lead Time × Safety Factor (uses supplier's LT and SF)

Suggested Order Qty:
• Normal items: max(MOQ, Max Level − Current Stock)
• Fast movers (velocity-triggered): last batch × multiplier (e.g. received 4 pcs → order 8)
Lead Time and Safety Factor are set per Supplier (one place, applies to all SKUs from that supplier).
